15 /***
16  * A fixed-size queue that releases its members only after a delay.
17  *
18  * This class forwards most of its functionality on to a
19  *champsim::circular_buffer<>, but introduces some boilerplate code to wrap
20  *members in a type that counts down the cycles until the member is ready to be
21  *released.
22  *
23  * The `end_ready()` member function (and related functions) are provided to
24  *permit iteration over only ready members.
25  *
26  * No checking is done when members are popped. A recommended way to operate on
27  *and to pop multiple members is:
28  *
29  * delay_queue<int> dq(10, 1);
30  * std::size_t bandwidth = 4;
31  * auto end = std::min(dq.end_ready(), std::next(dq.begin(), bandwidth));
32  * for (auto it = dq.begin(); it != end; it = dq.begin()) {
33  * perform_task(*it);
34  * dq.pop_front();
35  * }
36  *
37  * If no bandwidth constraints are needed, the terminal condition can simply be
38  *`it != dq.end_ready()`.
39  ***/
